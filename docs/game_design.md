# 游戏基础框架设计

## 项目定位

当前项目是一个 2D 字符地图动作游戏原型。现阶段已经有基础控制、地图显示和主循环，后续开发重点是补齐地图、技能、战斗、敌人、Boss、存档、NPC、经济和事件系统。

---

## 玩家基础属性

### 生命

- 血条上限：5-7 条
- 初始血量：5 条
- 可通过商人购买额外血量
- 最多额外购买 2 条血量

### 灵魂

- 数值范围：0-99
- 通过攻击、战斗等方式获取
- 用于释放法术和回血
- HUD 采用小型魂球容器样式：
  - 空：`/-\` + `|   |` + `\_/`
  - 半：`| o |`
  - 满：`|ooo|`

### HKD

- 数值范围：0-正无穷
- 作为经济系统货币使用
- 死亡后清零

## 玩家技能设计

### 基础攻击

1. 上劈
2. 下劈
3. 普攻

### 普攻收益

- 普攻触碰到敌人时：灵魂 +11

### 灵魂技能

1. 向左/右发射攻击波
   - 消耗：33 灵魂
2. 向上发射更强的攻击波
   - 消耗：33 灵魂
3. 向下释放法术重砸
   - 消耗：33 灵魂
4. 回血
   - 消耗：33 灵魂
   - 条件：需要连续 3 秒没有被敌人击中才可完成

### ASCII 回血动画实现方案

在纯 ASCII 限制下，回血动画优先通过“节奏感”而不是复杂图形来表现。

#### 实现思路

1. 灵魂容器脉冲
   - 回血施法期间，左上角灵魂容器内部填充字符周期切换
   - 推荐顺序：`@ -> * -> # -> *`
   - 作用：表现灵魂正在被抽取和转化

2. 玩家周围上升粒子
   - 在角色 `@` 周围生成向上飘动的小粒子
   - 推荐字符：`.` `:` `*` `'`
   - 粒子从角色两侧和头顶附近逐步上升

3. 血量球逐步点亮
   - 被恢复的那一格血量不要瞬间完成
   - 推荐顺序：`( ) -> (.) -> (:) -> (*)`
   - 作用：表现生命正在慢慢注入

4. 回血时锁定角色
   - 回血施法期间角色保持原地，不执行普通移动和攻击输入
   - 作用：增强“施法动作”的可读性

5. 回血完成时结算
   - 动画播放结束后才真正恢复 1 点血量
   - 灵魂消耗在施法开始时扣除
   - 如果后续加入打断机制，则动画中断后不恢复血量

### ASCII 向上攻击波动画方案

向上攻击波不要表现成普通竖直投射物，而要做成“聚魂 -> 冲柱 -> 炸冠 -> 消散”的节奏。

#### 实现思路

1. 蓄力
   - 在角色头顶先出现 `.^.`、`/ \` 这类短促聚魂符号
   - 作用：告诉玩家法术不是瞬发直线，而是先向上憋出一口魂力

2. 细柱上冲
   - 用 `!` 作为主伤害核心，向上拉出一根细柱
   - 可辅以 `.` 做弱残影
   - 作用：保留“向上吼叫”的贯穿感

3. 顶部炸冠
   - 在峰值帧使用 `\^^^/`、`***`、`\|/` 组成顶部爆裂轮廓
   - 作用：让法术在顶端有一次明显的能量释放，而不是只有一根线

4. 短暂消散
   - 收尾帧只保留少量 `.`、`^`、`|` 残痕
   - 作用：避免法术硬切消失

5. 字符语义统一
   - `!`：主伤害柱
   - `*`：爆裂核心
   - `^`：上冲尖端
   - `\|/`：外扩边缘
   - `.`：魂粒子残影

### ASCII 左右攻击波动画方案

左右攻击波适合做成克制、清晰、长度适中的基础法术弹道，不要做得像大招。

#### 实现思路

1. 出手帧
   - 向右：`-=>`
   - 向左：`<=-`
   - 作用：给一个短促的起手感

2. 飞行主帧
   - 向右：`~==>`
   - 向左：`<==~`
   - 可在主波体附近补少量 `.` 和 `'` 作为轻微波纹
   - 作用：让方向和速度一眼可读

3. 消散帧
   - 向右：`~~>`
   - 向左：`<~~`
   - 作用：收尾时不要硬切

4. 字符语义统一
   - `=`：主波体
   - `>` / `<`：前端尖头
   - `~`：尾流
   - `.` / `'`：轻微魂波纹

### ASCII 向下黑砸动画方案

向下黑砸最适合表现成“脚下聚气 -> 细柱下坠 -> 落地小范围爆开 -> 余波消散”，不要做得太宽。

#### 实现思路

1. 起手悬空
   - 在角色脚下出现 `.` 和 `\|/`
   - 作用：表现黑气正在脚下聚集

2. 下坠尖柱
   - 用连续 `!` 组成窄而直的下坠主柱
   - 作用：强调是向下钻地，不是大范围乱炸

3. 命中爆点
   - 命中地面时在落点用 `***` 做核心，并在两侧补 `<-` `->`
   - 作用：让落地一刻有明确重击感

4. 消散
   - 爆点下一帧只保留 `.` 与 `\|/`
   - 作用：保留一点黑气尾韵，但不要拖太长

5. 字符语义统一
   - `!`：下坠主伤害柱
   - `*`：落地爆点核心
   - `\|/`：冲击扩散边缘
   - `.`：残余黑气 / 余波

### ASCII 基础挥砍动画方案

基础挥砍优先做成短促、轻量、容易读的动作，不需要复杂多帧。

#### 实现思路

1. 左右劈
   - 向右：在角色前方显示 `-/`
   - 向左：在角色前方显示 `\-`
   - 作用：快速表达横向挥砍方向

2. 上劈
   - 先在头顶给出 `^`
   - 再展开成 `/|\`
   - 作用：一眼能读出向上抬刀

3. 下劈
   - 先在脚下给 `!`
   - 再落成 `v` 与 `!`
   - 作用：表现向下刺 / 下斩

4. 命中反馈
   - 敌人受击时暂时从 `g` / `f` 变成 `*`
   - 若死亡则转入死亡闪烁与消失

### 受伤反馈

- 玩家受伤后闪烁 2 秒
- 这 2 秒用于提供受击反馈

## 敌人设计

## 地面普通敌人

- 外观占位：`@`

### 属性

- 血条

### 行为

1. 左右移动
2. 玩家进入一定距离后，头顶触发 `!!!`
3. 预警后向玩家方向快速 Dash
4. Dash 算作一次攻击
5. 每次攻击完成后有 1 秒冷却
6. 玩家距离过远后停止追击，恢复普通状态

### 伤害

- 每次攻击造成 1 滴血

---

## 飞行敌人

### 属性

- 血条

### 行为

1. 上下移动
2. 攻击前摇（暂定）
3. 发射火球

### 伤害

- 每次攻击造成 1 滴血

## Boss 设计

目前有两个 Boss：

1. 近战 Boss
2. 远程 Boss

两者共通属性如下：

- 血量：100
- 硬直机制：累计打掉 40 滴血后进入硬直
- 硬直时长：5 秒
- 硬直期间玩家可以继续输出或回血

### 近战 Boss

#### 技能

1. 闪现到玩家背后 Dash 劈砍
2. 在玩家正面进行跳劈
3. 正面 Dash 冲刺攻击

---

### 远程 Boss

#### 技能

1. 法杖敲头
   - 靠近时造成玩家向后强制位移
2. 发射小火球
   - 玩家可以跳跃躲避
3. 陨石雨
   - 地面有颜色预警

#### 额外说明

- 远程 Boss 有普通攻击

### 掉落

2. 击杀 Boss：获得 50 HKD

### 死亡惩罚

## 游戏开始流程

进入游戏后，玩家可以：

1. 选择难度
2. 选择新存档或继续存档
3. 查看游戏通关时间

---

## 死亡与重生机制

### 死亡表现

- 显示：`YOU DEAD`

### 重生规则

- 玩家回到进入当前地图时的状态
- 恢复到该地图的初始保存点状态

恢复内容至少包括：

1. 血量
2. 游戏进度节点
3. HKD 状态
